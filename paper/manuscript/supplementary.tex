\section{Supplementary Note 1: Dataset and Import Pipeline Details}
% ---------------------------------------------------------------------------

\subsection{Rice 3K dataset}

The 3K Rice Genomes Project (Wang et al.\ 2018) SNP dataset comprises 3{,}024 accessions of Asian cultivated rice (\emph{Oryza sativa} L.) with $\sim$29.6 million SNPs called against the Nipponbare IRGSP-1.0 reference genome. Population classification follows Wang et al.\ (2018) ADMIXTURE at K=9, yielding 12 subpopulations: XI-1A, XI-1B, XI-2, XI-3, XI-adm (indica), GJ-tmp, GJ-trp, GJ-sbtrp, GJ-adm (japonica), cA-Aus, cB-Bas, and admix.

\subsection{Graph construction}

The import pipeline (Python, using cyvcf2) processes VCF files in a single streaming pass, producing CSV files for neo4j-admin bulk import. Genotypes are packed during streaming: \texttt{gt\_packed} (2 bits/sample, numpy-vectorized) and \texttt{phase\_packed} (1 bit/sample). Allele counts per population are pre-aggregated during import. VEP annotations and Reactome pathways are loaded by dedicated parsers. The resulting rice 3K graph database is $\sim$46~GB.

% ---------------------------------------------------------------------------
\section{Supplementary Note 2: Packed Genotype Encoding}
% ---------------------------------------------------------------------------

The \texttt{gt\_packed} byte array uses 2 bits per sample with encoding: 00=HomRef, 01=Het, 10=HomAlt, 11=Missing. The cyvcf2 internal encoding ($\{0, 1, 2, 3\}$) is remapped ($0 \to 0$, $1 \to 1$, $2 \to 3$, $3 \to 2$) to place Missing at the highest value. For 3{,}202 samples, each variant requires 801 bytes for genotypes and 401 bytes for phase (1 bit/sample). This encoding replaced a prior model using individual CARRIES edges ($\sim$6.3 billion for rice 3K), reducing the database from $\sim$200~GB to $\sim$46~GB.

The bit-packed HaplotypeMatrix used by FULL PATH procedures achieves further compression: each haplotype is 1 bit, packed 8 per byte. For 1{,}006 EUR samples (2{,}012 haplotypes), each variant row is 252 bytes versus 2{,}012 bytes unpacked---an 87\% reduction.

\section{Supplementary Note 3: EHH Computation Details}
% ---------------------------------------------------------------------------

EHHComputer walks position-sorted haplotype arrays from a focal variant outward (both directions), tracking distinct haplotype classes. The implementation uses flat array tracking with incremental sum-of-pairs updates (M1.4 optimization), replacing the earlier HashMap-per-step approach. Key configurable parameters: \texttt{min\_ehh} (EHH truncation threshold, default 0.05), \texttt{max\_gap} (maximum physical gap before aborting, default 200~kb), \texttt{gap\_scale} (distance cap for trapezoidal integration, default 20~kb). iHH is computed as the trapezoidal integral of the EHH curve.

For iHS and XP-EHH, chromosomes are processed in chunks (5~Mb core window + 2~Mb EHH margin on each side) to bound peak memory at $\sim$200~MB per chunk. Standardization (AF-bin for iHS/nSL; genome-wide for XP-EHH) is applied across all chunks after computation.

% ---------------------------------------------------------------------------
\section{Supplementary Note 4: ROH HMM Details}
% ---------------------------------------------------------------------------

The ROH HMM implements a two-state (Autozygous/Hardy--Weinberg) Viterbi decoder following Narasimhan et al.\ (2016). Transition probabilities use exact 2-state continuous-time Markov chain (CTMC) matrix exponentiation: $P(\text{HW} \to \text{AZ}|d) = \pi_\text{AZ} \cdot (1 - (1-\alpha-\beta)^d)$, where $d$ is the inter-variant distance in base pairs. Emission probabilities are allele-frequency weighted: $P(\text{hom}|\text{AZ}) = 1 - \epsilon$ (with het error rate $\epsilon = 10^{-3}$), $P(\text{hom}|\text{HW}) = p^2 + (1-p)^2$. Default per-bp rates match bcftools roh -G30 (\texttt{hw\_to\_az} = $6.7 \times 10^{-8}$, \texttt{az\_to\_hw} = $5 \times 10^{-9}$). Validation: $r = 0.96$ per-sample total ROH length vs bcftools on 1000 Genomes EUR chr22.

% ---------------------------------------------------------------------------
\section{Supplementary Note 5: Human 1000 Genomes Full-Genome Results}
% ---------------------------------------------------------------------------

In addition to the chr22 benchmarks, GraphPop was applied to the full 1000 Genomes Phase~3 dataset across all autosomes (chr1--22) for all 5 superpopulations. Genome-wide diversity, divergence, genome scans, iHS, XP-EHH, and nSL results are available in the pre-built graph database deposited at Figshare/Zenodo. Per-population genome-wide summary statistics (Supplementary Table~\ref{tab:human_summary}) confirm expected patterns: AFR has the highest diversity, EAS and EUR show stronger LD, and known selection signals (e.g., \emph{LCT} region in EUR, \emph{SLC24A5} in EUR) are recovered.
