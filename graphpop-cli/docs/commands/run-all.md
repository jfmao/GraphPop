# graphpop run-all

## Title

Full-Genome Orchestration Across All Populations and Chromosomes

## Description

Runs the complete GraphPop analysis pipeline across all populations and
chromosomes in the database. Executes in two phases: Phase 1 computes
per-population statistics (diversity, SFS, iHS, nSL, ROH, Garud's H), and
Phase 2 computes pairwise statistics (XP-EHH, divergence). Supports resuming
interrupted runs, progress reporting, and JSON checkpoint files.

## Usage

```
graphpop run-all [OPTIONS]
```

## Arguments

This command takes no positional arguments.

## Options

| Option | Type | Default | Description |
|---|---|---|---|
| `--phase` | CHOICE | `all` | Which phase to run: `1` (per-pop), `2` (pairwise), or `all`. |
| `-d`, `--output-dir` | PATH | `graphpop_results` | Output directory for TSV result files, organized by procedure. |
| `--resume` / `--no-resume` | FLAG | `--resume` | Skip tasks already recorded in the checkpoint JSON. Enabled by default. |
| `--json-output` | PATH | `<output-dir>/results.json` | Path to the JSON checkpoint file. |
| `--persist` / `--no-persist` | FLAG | `--persist` | Write results to graph nodes. Enabled by default. |
| `--populations` | TEXT | *(auto-detect)* | Comma-separated list of population IDs. If omitted, queries all populations with >1 sample from the graph. |
| `--chromosomes` | TEXT | *(auto-detect)* | Comma-separated list of chromosome IDs. If omitted, queries all chromosomes from the graph. |
| `--xpehh-pairs` | INT | `20` | Maximum number of population pairs for XP-EHH in Phase 2. |
| `--workers` | INT | `1` | Number of parallel workers (experimental). |

## Value

  diversity/

## Details

### Phase 1: Per-Population Statistics

For each `population x chromosome` combination, runs:

| Procedure | Type | Description |
|---|---|---|
| `diversity` | Fast path | pi, theta_W, Tajima's D, Fay & Wu's H, H_e, H_o, F_IS |
| `sfs` | Fast path | Site frequency spectrum |
| `pop_summary` | Fast path | Whole-chromosome summary statistics |
| `roh` | Full path | Runs of homozygosity per sample |
| `ihs` | Full path | Integrated haplotype score (min AF 0.05) |
| `nsl` | Full path | Number of segregating sites by length (min AF 0.05) |
| `garud_h` | Full path | Garud's H1, H12, H2/H1 in 100 kb windows (50 kb step) |

Total tasks = `n_populations x n_chromosomes x 7`.

### Phase 2: Pairwise Statistics

For each population pair (up to `--xpehh-pairs` pairs) and chromosome, runs:

| Procedure | Type | Description |
|---|---|---|
| `xpehh` | Full path | Cross-population extended haplotype homozygosity (min AF 0.05) |
| `divergence` | Fast path | Hudson Fst, W&C Fst, Dxy, Da, PBS |

Population pairs are generated by taking all unique combinations of the
detected populations, sorted by sample count (largest first), capped at the
`--xpehh-pairs` limit.

### Auto-Detection

When `--populations` and `--chromosomes` are not specified, the command queries
the graph:

- **Populations**: `MATCH (p:Population) WHERE p.n_samples > 1 RETURN p.populationId ORDER BY p.n_samples DESC`
- **Chromosomes**: `MATCH (c:Chromosome) RETURN c.chromosomeId, c.length`

Chromosome lengths are used to set the correct region boundaries for fast-path
procedures.

### Resume Support

The JSON checkpoint file (`results.json`) tracks every completed task by a key
like `EUR_chr22_diversity` or `EUR_vs_AFR_chr22_xpehh`. When `--resume` is
enabled (the default), tasks already present in the checkpoint are skipped.

The checkpoint is saved after completing each chromosome, ensuring minimal
re-work if the run is interrupted. The save uses atomic file replacement
(write to `.tmp`, then rename) to prevent corruption.

### Progress Reporting

Every 10 tasks, the command prints progress:

```
[70/1120] EUR_chr3_ihs (145s elapsed, ~2196s remaining)
```

### Error Handling

If a procedure call fails, the error is recorded in the JSON checkpoint under
the task key and the run continues with the next task. At the end, a summary
reports the number of successes and failures.

## Examples

### Full analysis with defaults

```bash
graphpop run-all -d results/
```

### Phase 1 only for specific populations

```bash
graphpop run-all \
    --phase 1 \
    --populations EUR,AFR,EAS \
    -d results/
```

### Phase 2 only with custom pair limit

```bash
graphpop run-all \
    --phase 2 \
    --xpehh-pairs 10 \
    -d results/
```

### Resume an interrupted run

```bash
# First run (interrupted)
graphpop run-all -d results/
# Ctrl-C or crash...

# Resume from where it left off
graphpop run-all -d results/ --resume
```

### Single chromosome, all populations

```bash
graphpop run-all \
    --chromosomes chr22 \
    -d results_chr22/ \
    --json-output results_chr22.json
```
