🧬 GraphPop Platform

Integrative Evolutionary Inference from Population Variation Graphs

Revised Implementation Roadmap v3.0 — Unified from Design v1, v2, and original roadmap. Targets All of Us scale.

Project Overview

Core Vision

GraphPop transforms population genomics by treating variant datasets as living, queryable graphs where structure, recombination, selection, and evolutionary dynamics become different projections of the same relational object. It targets datasets from rice 3K to the NIH All of Us program (1 billion+ variants, 1 million+ samples).

🎯 Primary Goal

Replace isolated VCF-centric pipelines with a unified graph database enabling seamless integration of all population genetic analyses — at any scale from laptop to cloud cluster.

💡 Innovation

Graph embeddings + GraphRAG + AI agents create feedback loops impossible in classical workflows. Dual-path compute engine (fast allele-count path + sparse CARRIES path) enables million-sample analyses.

🌾 Target Systems

Crops (rice, maize, wheat) and their wild relatives AND human populations via All of Us (500K+ genomes). Both evolutionary inference and biomedical applications.

Two Foundational Requirements

🔴 Requirement 1: All of Us Scale

1+ billion variants, 1+ million samples, 100+ TB raw genotype data. From NIH All of Us: 414,830 srWGS (Feb 2025), 535K planned spring 2026, 1M+ target.

🔵 Requirement 2: Fully Graph-Native

No external columnar store. The graph database IS the computational substrate. All genotype data, statistics, and inferences live in Neo4j.

Why Graph-Native Matters

  • Memory Efficiency: Window queries load only relevant data (5–100× less RAM)
  • Speed: Fast-path statistics O(V×K) independent of sample count N; ~20,000× less I/O than dosage vectors at 1M samples
  • Composability: Combine FST, Tajima's D, recombination rate, and functional enrichment in single queries
  • Evolvability: Incremental updates — new sample adds ~4.5M CARRIES edges without touching any Variant nodes
  • Relational Queries: "Find all individuals sharing an IBD segment overlapping a high-FST missense variant in a drought-response pathway" — one query, no ad-hoc pipeline

System Architecture

Seven-Layer Architecture

Layer 7: AI-Augmented Inference (GraphRAG, LangGraph agents, NL interface) Layer 6: Simulation Integration (msprime/SLiM, ABC framework) Layer 5: Inference Engines (demographic, selection, recombination) Layer 4: Graph Algorithms & Embeddings (community detection, GNN, manifolds) Layer 3: Pairwise & Relational Statistics (kinship, IBS/IBD, ROH) Layer 2: Summary Statistics Engine (SFS, π, FST, iHS — FAST + FULL paths) Layer 1: Data Foundation (schema, import/export, VectorOps SIMD, CARRIES)

The Stratified Aggregation Model (Schema v3)

Node Types

  • Variant: Population allele count arrays (ac, an, af, het_count per pop), ancestral state, functional annotation summary, quality
  • Sample: Individual metadata, ancestry, PCA coords, phenotypes, embeddings
  • Population: Grouping + pre-computed constants (an, an2, bn for Tajima's D)
  • Chromosome: Length, variant count, centromere coordinates
  • GenomicWindow: Materialized windowed statistics (all diversity/divergence metrics)
  • DenseWindow: On-demand cached genotype matrix for common variants (LD, PCA)
  • Gene / RegulatoryElement / Pathway / GOTerm: Functional annotation layer
  • InferenceResult: Versioned outputs from inference methods

Relationship Types

  • NEXT: Positional chain per chromosome (distance_bp, distance_cM)
  • CARRIES: THE core relationship — sparse Sample→Variant (gt: 1=HET, 2=HOM_ALT; phase). HOM_REF implicit.
  • LD: Linkage disequilibrium (r², D', population). Sparse, above threshold.
  • ON_CHROMOSOME: Variant→Chromosome
  • HAS_CONSEQUENCE: Variant→Gene with VEP consequence type + impact
  • IN_PATHWAY / HAS_GO_TERM / IS_A: Functional annotation hierarchy
  • IN_POPULATION: Sample→Population membership
  • KINSHIP: Sparse Sample→Sample (above threshold)
  • IBD_SEGMENT: Sample→Sample shared haplotype segments
  • COMPUTED_FROM / CONDITIONED_ON: Inference provenance

Dual-Path Computation Engine

⚡ Fast Path (O(V×K))

Uses ONLY allele count arrays on Variant nodes. No CARRIES traversal.

  • π (nucleotide diversity)
  • θW (Watterson's theta)
  • Tajima's D, Fay & Wu's H
  • He, Ho, FIS
  • FST (Hudson), Dxy
  • SFS, Joint SFS
  • genome_scan() (windowed)

🔍 Full Path (CARRIES traversal)

Traverses sparse CARRIES edges. Efficient for rare variants (>90% at AoU scale).

  • r² / D' (LD)
  • iHS, XP-EHH
  • Kinship (KING-robust)
  • PCA / GRM construction
  • IBD segment detection
  • ROH detection

📊 Scale: From Laptop to All of Us

The All of Us Data Challenge

All of Us (NIH): Current & Projected
  • 245,388 WGS (Nature 2024) → 414,830 srWGS (Feb 2025) → 535K (spring 2026) → 1,000,000+ target
  • >1 billion variants from 245K genomes; projected 1.5–2B at 1M genomes
  • Dosage-vector model: 1B variants × 1M samples × 1 byte = 1 petabyte — impossible
  • Stratified Aggregation Model: ~140 TB total — feasible with Infinigraph cluster

Three-Tier Deployment

Tier 1: Development

  • i9-13900K, 64 GB, RTX 4090, 4 TB NVMe
  • Single chromosome, 5K–50K samples
  • Neo4j Community 5.x on WSL2
  • ~50M variants, 200M–1B CARRIES
  • 5–140 GB total

Tier 2: Production Research

  • 3–5 machine cluster (cloud or on-prem)
  • Whole genome, 50K–200K samples
  • Neo4j Enterprise + Infinigraph
  • ~1B variants, 200B–500B CARRIES
  • 10–50 TB — publishable paper scale

Tier 3: Full All of Us

  • 10–20+ machine cluster
  • Whole genome, 500K–1M+ samples
  • Neo4j Enterprise Infinigraph
  • ~1.5B variants, 4.5 trillion CARRIES
  • 100–300 TB — long-term target

Key Principle

Schema, stored procedures, and query patterns are IDENTICAL across all three tiers. Only data volume and deployment topology change. Code written on your laptop works unchanged on the full cluster.

Build → Validate → Discover → Publish. Each publication is tightly synchronized with completed implementation. Sequential phases with minimal overlap ensure quality. This roadmap integrates the original design milestones, the All of Us scale revisions, and the publication timeline.

Novel Applications & Theory

1. Graph-Derived Summary Statistics

Community structure coefficients, embedding dimensionality, manifold curvature, centrality distributions — new statistics beyond SFS/LD. Rigorously testable with simulation-based power analysis.

2. Conditioned Statistics as Default

Any statistic conditioned on functional annotation, recombination rate, or population structure in a single query. Making conditioned computation routine changes interpretation of diversity patterns.

3. Recombination Topology Bottleneck Hypothesis

In recombination coldspots, the variant graph has long path lengths and high betweenness centrality → predicts stronger selective interference.

Test: Compare sweep signatures in high vs low recombination subgraphs.

4. Accessory Recombination Reservoirs (Crop Evolution)

Accessory variants in pan-genome graphs have higher recombination rates → a reservoir of standing variation for rapid adaptive combinations under climate stress.

5. Selection Compression Manifolds

Hard sweeps compress local embedding manifolds; balancing selection creates ridges; purifying selection creates corridors. A new statistic predicting selection strength and age.

6. Functional Variant Connectivity Hotspots

Accessory genome variants with many short functional paths to core genes (high betweenness) are under recent positive selection — modulating conserved pathways without disrupting core function.

7. CARRIES Bipartite Graph as Population-Genetic Object v3 Novel

SFS = degree distribution of Variant nodes. Heterozygosity = degree of Sample nodes. IBS = shared neighborhoods. Population structure = bipartite community structure. Making explicit what was always implicit in the genotype matrix.

8. GraphRAG-Powered Evolutionary Laboratories

Iterative: GraphRAG retrieves "Which variants are in high-recombination regions and show elevated FST?" → Agent proposes selection coefficients → Runs SLiM simulation → Re-queries → Iterates until convergence.
